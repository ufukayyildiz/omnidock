What OmniDock runs

OmniDock is designed for teams that want a self-hosted email operations console on Cloudflare rather than a hosted SaaS inbox. It keeps secrets in Worker configuration, stores operational data in D1, keeps raw messages and attachments in R2, and exposes a focused admin UI for day-to-day support and file workflows.

1. Bind Connect D1, R2, Email Sending, and required Worker secrets.
2. Configure Add domains, mailbox addresses, contacts, signatures, and optional Gmail accounts.
3. Sync Pull external messages, sync Cloudflare routing state, and log operational actions.
4. Operate Reply, archive, search, inspect attachments, and manage R2 bucket files.

Gmail and external email accounts

Connect external accounts as operator mailboxes without storing real passwords in D1. Gmail app passwords, Outlook credentials, Yahoo credentials, iCloud app passwords, or custom provider secrets live in Cloudflare Worker secrets. OmniDock stores only the provider, email address, IMAP/SMTP host settings, and the secret reference.

  • Gmail preset with IMAP over SSL and SMTP over SSL.
  • Outlook, Yahoo, iCloud, and custom IMAP/SMTP presets.
  • Inbound sync, outbound send, or both per account.
  • D1-backed sync jobs so refreshing the dashboard does not cancel long pulls.
  • Per-run progress and logs for imported, skipped, checked, and timed-out messages.

Cloudflare domains and mailbox rules

OmniDock syncs Cloudflare zones and helps operators understand which domains can send, receive, route a single mailbox, or accept catch-all mail. It is built for teams that manage many support addresses across many domains.

  • Cloudflare Email Sending verification state.
  • Email Routing status and catch-all status.
  • Mailbox-specific Worker routing rules.
  • Default domain and default mailbox selection.
  • Clear setup warnings for missing bindings, variables, and secrets.

R2 bucket manager with upload and preview

The Buckets area turns one primary mail bucket and any extra configured R2 buckets into a practical file workspace. Operators can switch buckets from a dropdown, browse folder prefixes, preview common file types, upload batches with visible progress, and delete objects through in-app confirmations.

  • PreviewImages, PDFs, and text files open inside the dashboard before download.
  • UploadBatch uploads show progress, current file state, and completion status.
  • SearchSearch this bucket or all buckets by filename, path, text file content, PDF text, and saved indexes.
  • ControlDownload and delete objects from the same preview workflow.

OCR-ready document search without surprise AI cost

OmniDock does not automatically run paid AI/OCR on every search. It searches extractable PDF text and stored text indexes. For scanned PDFs or images, open the object, choose Index text, paste OCR text from your preferred OCR tool, and OmniDock stores that index in D1 for future searches.

  • Search text files and PDFs with embedded selectable text.
  • Save OCR text manually for scanned PDFs, screenshots, labels, and image documents.
  • Keep repeated searches fast by storing the text index in D1.
  • Log slow or failed searches so operators can narrow scope or turn off text search.
Private by design. OmniDock is fork-first and self-hosted. Your D1 database id, R2 bucket names, API tokens, admin password, app passwords, custom domains, and personal email addresses should live in your own Cloudflare settings or private fork, not in the public repository.

Install on Cloudflare Workers

Fork the repository, create D1 and R2 resources, deploy from your fork, set the binding-safe deploy command, add runtime secrets, and open your Worker URL. The README includes the full install guide, required bindings, Cloudflare API token permissions, extra R2 bucket configuration, and first-login flow.
